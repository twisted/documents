Part of twisted.web.woven.model View Source View In Hierarchy

I may wrap an object and allow it to interact with the Woven models and submodels. By default, I am not registered for use with anything. I am unsafe because I allow methods to be called. In fact, I am dangerously unsafe. Be wary or I will kill your security model!
Method getSubmodel Get the submodel `name' of this model. If I ever return a Deferred, then I ought to check for cached values (created by setSubmodel) before doing a regular Deferred lookup.
