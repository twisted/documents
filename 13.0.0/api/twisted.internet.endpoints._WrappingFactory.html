ParametersdeferredThe Deferred that was cancelled; should be the same as self._onConnection. (type: Deferred)

NoteThis relies on startedConnecting having been called, so it may seem as though there's a race condition where _connector may not have been set. However, using public APIs, this condition is impossible to catch, because a connection API (connectTCP/SSL/UNIX) is always invoked before a _WrappingFactory's Deferred is returned to connect()'s caller.
